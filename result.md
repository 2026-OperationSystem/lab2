# xv6 프로젝트 Git Commit 내역 조사 결과

본 프로젝트의 Git commit 내역을 조사한 결과, 주된 변경 사항은 xv6 운영체제에 새로운 시스템 콜(`exit2`, `wait2`)을 구현하고 새로운 사용자 프로그램(`team1id`)을 추가하는 과제를 수행한 것입니다. 주요 변경 사항과 수정 이유는 다음과 같습니다.

## 1. `exit2` 및 `wait2` 시스템 콜 구현
- **수정 목적**: 기존 xv6의 `exit`와 `wait` 시스템 콜은 종료 상태(exit status)를 전달하는 기능이 없습니다. 부모 프로세스가 자식 프로세스의 종료 상태를 확인할 수 있도록 상태 값을 전달받는 `exit2`와 `wait2` 시스템 콜을 새롭게 구현하여 기능을 확장하기 위함입니다.
- **주요 수정 파일 및 내용**:
  - `xv6edk2/xv6/proc.h`: `struct proc` 구조체에 자식 프로세스의 종료 코드를 임시로 저장할 `int xstate;` 변수를 추가했습니다.
  - `xv6edk2/xv6/proc.c`: 종료 코드를 받아 `xstate`에 저장하고 프로세스를 `ZOMBIE` 상태로 전환하는 `exit2(int status)` 함수와, 좀비 상태인 자식 프로세스를 찾아 `xstate` 값을 포인터를 통해 반환하는 `wait2(int *status)` 함수를 구현했습니다.
  - `xv6edk2/xv6/syscall.h`, `syscall.c`, `sysproc.c`: 시스템 콜 번호(`SYS_exit2`, `SYS_wait2`)를 할당하고 커널 모드에서 이를 처리할 래퍼(wrapper) 함수(`sys_exit2`, `sys_wait2`)를 추가했습니다.
  - `xv6edk2/xv6/user.h`, `usys.S`: 유저 공간에서 응용 프로그램들이 해당 시스템 콜을 호출할 수 있도록 인터페이스를 선언했습니다.
  - `xv6edk2/xv6/testwait.c`: 구현한 시스템 콜이 정상적으로 동작하는지 테스트하기 위해, 자식 프로세스가 `exit2`로 코드를 반환하고 부모 프로세스가 `wait2`를 통해 상태 코드를 출력하는 테스트 프로그램을 추가했습니다.

## 2. 사용자 프로그램 `team1id` 추가
- **수정 목적**: xv6 운영체제 내에서 실행할 수 있는 새로운 사용자 명령어(User Program)를 추가하는 방법을 실습하기 위함입니다.
- **주요 수정 파일 및 내용**:
  - `xv6edk2/xv6/team1id.c`: 실행 시 "My ID is 202201537"이라는 학번 정보를 출력하고 종료하는 유저 레벨 소스 코드를 작성했습니다.
  - `xv6edk2/xv6/Makefile`: 컴파일 타겟인 `UPROGS` 목록에 `_team1id`를 추가하여, xv6를 빌드할 때 `team1id` 명령어가 파일 시스템 이미지(`fs.img`)에 정상적으로 포함되도록 설정했습니다.

## 3. 프로젝트 초기화 및 빌드 아티팩트 정리
- **수정 목적**: 초기 xv6 소스코드 구성 및 깃 레포지토리의 용량을 관리하고 불필요한 파일 충돌을 방지하기 위함입니다.
- **주요 수정 내용**: 프로젝트 초기 구성(`defaultSetting`) 이후, `.o`, `.d`, `.asm`, `.sym` 및 여러 바이너리 파일 등 빌드 과정에서 생성되는 부산물들을 대거 삭제(`feat/빌드된거 삭제` 커밋)하여 소스 코드 위주로 레포지토리를 정리했습니다. 또한 `README.md`를 작성하여 프로젝트 관련 설명을 추가했습니다.
